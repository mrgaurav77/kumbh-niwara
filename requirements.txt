# KumbhNiwara Dependencies

While this is a strictly Node.js (JavaScript) and React based project that utilizes `package.json` for dependency management instead of `requirements.txt`, here is a comprehensive list of the core dependencies and their versions utilized in both the Backend and Frontend to run the application. 

You can install these dependencies using `npm install` in their respective directories.

---

## Backend Dependencies (Node.js/Express)
Located in `/backend` folder.

- **express**: (^5.2.1) - Fast, unopinionated, minimalist web framework.
- **mongoose**: (^9.4.1) - MongoDB object modeling tool.
- **cors**: (^2.8.6) - Express middleware to enable CORS.
- **dotenv**: (^17.4.1) - Loads environment variables from a .env file.
- **bcrypt**: (^6.0.0) - Library to help hash passwords for user authentication.
- **jsonwebtoken**: (^9.0.3) - JSON Web Token implementation to authorize API requests.
- **multer**: (^2.1.1) - Middleware for handling `multipart/form-data`, primarily used for image uploads.

## Frontend Dependencies (React)
Located in `/my-app` folder.

**Core Web Packages:**
- **react**: (^19.2.4) - A JavaScript library for building user interfaces.
- **react-dom**: (^19.2.4) - Serves as the entry point to the DOM and server renderers for React.
- **react-router-dom**: (^7.13.2) - DOM bindings for React Router (navigating between pages).
- **react-scripts**: (5.0.1) - Includes scripts and config used by Create React App.

**UI / Styling:**
- **tailwindcss**: (^3.4.19) - A utility-first CSS framework.
- **postcss**: (^8.5.8) - Tool for transforming CSS with JavaScript.
- **autoprefixer**: (^10.4.27) - PostCSS plugin to parse CSS and add vendor prefixes.
- **lucide-react**: (^0.577.0) - Beautiful & consistent icons library.

**Testing:**
- **@testing-library/react**: (^16.3.2)
- **@testing-library/jest-dom**: (^6.9.1)
- **@testing-library/dom**: (^10.4.1)
- **@testing-library/user-event**: (^13.5.0)

**Web Vitals:**
- **web-vitals**: (^2.1.4)
